# Менеджер рецептів Євгена Клопотенка

Це моя лабораторна робота на Spring Boot.  
Я зробила веб-застосунок для збереження та перегляду рецептів, натхненний рецептами Євгена Клопотенка.

У застосунку можна:
- додавати рецепти вручну
- переглядати всі рецепти
- редагувати рецепти
- видаляти рецепти
- працювати з категоріями
- імпортувати рецепт із сайту Євгена Клопотенка за посиланням

Проєкт зроблений як клієнт-серверний застосунок на Java з використанням Spring Boot, бази даних та HTML-шаблонів.

---

## Що це взагалі за проєкт

Ідея проєкту проста: я хотіла зробити менеджер рецептів, де можна зберігати рецепти в одному місці, групувати їх по категоріях і додавати інгредієнти.  
Також я реалізувала окремий модуль імпорту, який парсить рецепт із зовнішнього сайту через Jsoup.

Тобто це не просто список рецептів, а повноцінний CRUD-застосунок з базою даних, REST API і веб-інтерфейсом.

---

## Основний функціонал

У проєкті реалізовано:

- CRUD для рецептів
- CRUD для категорій
- збереження інгредієнтів для кожного рецепта
- REST API для роботи з даними
- веб-інтерфейс на Thymeleaf
- імпорт рецепта з сайту `klopotenko.com`
- обробка помилок через глобальний exception handler

## Структура проєкту

Проєкт побудований за стандартною багатошаровою структурою:

- `entity` — сутності бази даних
- `repository` — доступ до даних через JPA

## Основні сутності

У проєкті є 3 основні сутності:

### 1. Category
Категорія рецепта.  
Наприклад:
- супи
- десерти
- основні страви

### 2. Recipe
Основна сутність, яка зберігає інформацію про рецепт:
- назва
- опис
- інструкція приготування
- час приготування
- кількість порцій
- зображення
- категорія

### 3. Ingredient
Інгредієнт рецепта:
- назва
- кількість
- прив’язка до рецепта

---

## Зв’язки між сутностями

У базі даних реалізовано такі зв’язки:

- одна категорія може містити багато рецептів
- один рецепт може містити багато інгредієнтів

Тобто в проєкті використано зв’язки типу:
- `One-to-Many`
- `Many-to-One`

## REST API

У застосунку є REST API для роботи з даними.

### Категорії
- `GET /api/categories` — отримати всі категорії
- `GET /api/categories/{id}` — отримати категорію за id
- `POST /api/categories` — створити категорію
- `PUT /api/categories/{id}` — оновити категорію
- `DELETE /api/categories/{id}` — видалити категорію

### Рецепти
- `GET /api/recipes` — отримати всі рецепти
- `GET /api/recipes/{id}` — отримати рецепт за id
- `POST /api/recipes` — створити рецепт
- `PUT /api/recipes/{id}` — оновити рецепт
- `DELETE /api/recipes/{id}` — видалити рецепт

Додатково реалізовано:
- пошук рецепта по назві
- фільтрація рецептів по категорії

Для зручності я також зробила веб-частину через Thymeleaf.

Є такі сторінки:
- список усіх рецептів

## Імпорт рецептів з сайту Євгена Клопотенка

Окремо реалізовано модуль парсингу через бібліотеку **Jsoup**.

Що він робить:
1. Користувач вставляє посилання на рецепт із сайту Євгена Клопотенка.
2. Застосунок завантажує HTML-сторінку.
3. Із неї витягуються основні дані:
   - назва рецепта
   - опис
   - зображення
   - час приготування
   - кількість порцій
   - інструкція
   - інгредієнти
4. Після цього рецепт автоматично зберігається в базу даних.

Для імпортованих рецептів автоматично створюється категорія **"Імпортовані рецепти"**, якщо її ще не було.

## База даних

У проєкті використовується **H2 Database**.  
Я вибрала її, бо вона проста для лабораторної роботи і не потребує окремого встановлення PostgreSQL чи MySQL.

Дані зберігаються у таких таблицях:
- `categories`
- `recipes`
- `ingredients`

Також можна відкрити консоль H2 в браузері.

---

## Як запустити проєкт

1. Відкрити проєкт в IntelliJ IDEA
2. Переконатися, що встановлено JDK 17
3. Запустити головний клас:
   `KlopotenkorecipemanagerApplication`
4. Після запуску відкрити браузер:
   `http://localhost:8080`

### H2 Console
- URL: `http://localhost:8080/h2-console`
- JDBC URL: `jdbc:h2:mem:recipesdb`
- Username: `sa`
- Password: порожній
